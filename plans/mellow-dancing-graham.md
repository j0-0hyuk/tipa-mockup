# 채팅창 멀티턴 대화 목업

## Context
현재 chatbot-flow 페이지는 인사 메시지 → 시나리오 칩으로 바로 시작하는데, 이미 사용자와 대화가 진행된 멀티턴 상태를 보여주도록 변경한다. TIPANI의 맥락 이해 능력과 자연스러운 대화 흐름을 시연하기 위함.

## 수정 파일
- `apps/web/src/routes/_authenticated/chatbot-flow/-components/script.ts`

## 대화 흐름

`INTRO_SCRIPT`를 아래 멀티턴 대화로 교체:

1. **TIPANI** (ai): 기존 인사 — "안녕하세요 👋 저는 R&D 지원사업 전반을 돕는 TIPANI예요. 계획서 작성부터 규정·증빙·예산 편성까지 도와드릴 수 있어요. 어떤 도움이 필요하신가요?"
2. **김민수** (user): "우리 회사 3년차, 매출 10억인데 지금 지원 가능한 사업 있어?"
3. **TIPANI** (ai): "🧠 **분석 중입니다** SMTECH 공고에서 업력·매출·졸업제 룰을 대조해드릴게요."
4. **TIPANI** (ai, instant): 지원사업 Top 3 추천 (기존 recommend 시나리오 답변 재사용)
5. **김민수** (user): "1번 창업성장기술개발사업 신청 마감이 언제야?" — 4번 답변의 ①을 참조한 후속 질문
6. **TIPANI** (ai): "🧠 **분석 중입니다** 창업성장기술개발사업 공고 일정을 확인해드릴게요."
7. **TIPANI** (ai, instant): 마감일·일정 상세 답변 (새로 작성)
8. **chips** — 추가 시나리오 선택

## 구현 세부사항
- 1번: 기존 INTRO_SCRIPT의 인사 메시지 유지
- 2~4번: 기존 `recommend` 시나리오의 스텝 재사용
- 5~7번: 후속 질문/답변 새로 작성
- 모든 메시지에 `instant: true` + 짧은 `delayBeforeMs`로 이미 진행된 대화처럼 빠르게 표시
- `SCENARIOS`의 `recommend`는 유지 (칩 선택 시 동작)

## 검증
- `pnpm web` → `/chatbot-flow` 접속
- 위젯 열면 1~7번 대화가 순서대로 빠르게 표시되는지 확인
- 마지막에 칩 카드가 정상 표시되는지 확인
- 칩 선택 시 기존 시나리오가 정상 동작하는지 확인
